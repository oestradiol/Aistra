# Project purpose and use cases

Aistra is a small prototype language and test package for describing felt states in a clearer and more structured way.

It breaks a state into nine parts:
- valence
- arousal
- intensity
- location
- texture
- contour
- certainty
- source
- action tendency

What it is useful for right now:
- structured introspection
- pilot testing of a state-description schema
- compact annotation of subjective-state reports
- interface and agent experiments that need a small felt-state representation

What it does not claim:
- it is not a validated clinical or psychometric instrument
- it is not a final theory of human feeling
- it is not proof that every person experiences states in this exact structure

Current package posture:
- Route 1 = operationally pilot-ready as a package/workflow label, one initial human pilot completed, not broadly validated
- Route 2 = pre-human
