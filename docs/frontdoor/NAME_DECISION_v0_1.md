# Name Decision v0.1

## Decision

The project and language name is now **Aistra**.

## Why this name

The package is about structured experiential reading and reporting. The chosen name aims to sound:

- compact
- human-readable
- distinct from generic AI-product naming
- compatible with a language, project, and interface identity

## Naming rule

Use:

- **Aistra** = project name
- **Aistra language** = the structured language itself
- **Route 1** and **Route 2** = internal route names retained for bounded package control

## Scope of this decision

This is a naming decision, not a full package-path rename.

Internal file and folder names remain on the current lineage to avoid avoidable churn and path breakage during the handoff phase.

## Future rename rule

Do not mass-rename the package tree unless there is a dedicated path-migration pass with validator coverage.
