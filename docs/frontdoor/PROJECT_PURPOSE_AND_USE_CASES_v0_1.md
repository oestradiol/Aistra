# Project Purpose and Use Cases v0.1

## What this project is

**Aistra** is a bounded prototype language and pilot instrument for encoding short human experiential states into a fixed structured form.

The current core representation uses nine scored fields:

- valence
- arousal
- intensity
- location
- texture
- contour
- certainty
- source
- action

Route 1 tests whether people can learn and use that representation reliably on brief state descriptions.

## What the point is

This project exists to make subjective-state reports more inspectable.

Instead of a single vague report like "it felt bad and weird," the package forces the report into separable dimensions that can be checked, scored, compared, audited, and improved.

That makes three things possible:

1. **structured self-report**
2. **error-localized human testing**
3. **bridge layers between plain language and formal state annotation**

## What it may be useful for

Current plausible uses:

- pilot research on structured experiential annotation
- calibration tasks for introspective labeling
- human/agent interface experiments where natural-language state reports need a compact formal target
- journaling or reflective tools that benefit from factorized state descriptions
- future dataset creation where each state report must be decomposable and scorable

## What it is not yet

This package is **not** currently:

- a clinically validated instrument
- a general theory of feeling
- a completed psychometric scale
- proof that the ontology is final
- proof that naive users will generalize well without training

## Current strongest contribution

The strongest current contribution is practical rather than grand:

**the package gives a compact structured target that makes human-report failure modes visible and repairable.**

## Current proof status

What is currently supported inside this package:

- the ontology is explicit
- the Route 1 structured pilot path is runnable
- the local interface reduces operator drift
- the package can audit important parts of itself
- at least one real human pilot has been completed and used to improve the package

What still requires more evidence:

- multi-user reliability
- broader external validity
- field necessity under stronger data
- stability across less guided participants
