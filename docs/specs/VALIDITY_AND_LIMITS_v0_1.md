# Validity and limits v0.1

## Current claim
Aistra currently supports a modest claim:
**Route 1 is a usable, scientifically grounded prototype that humans can learn and apply under controlled administration.**

## What supports that claim
- the field choices and route design are documented in the research-grounding layer
- participant performance improved rapidly once the ontology stabilized
- structured items became reliably decodable
- protocol weaknesses were diagnosable and repairable
- a local interface removes a large part of operator paraphrase drift

## What is still missing
- broader multi-user evidence
- test-retest and inter-rater evidence
- stronger evidence for external validity
- any basis for calling this a finished instrument

## Residual risks that still matter
- current Route 1 performance may still partly reflect packet-specific fit rather than broad representational generalization
- the 9-field ontology is consciously provisional and can still contain discretization error
- the interface can reduce operator paraphrase drift while still shaping answers through wording, layout, examples, or fatigue
- the current adversarial suite covers known failure classes better than open-world human weirdness
- strong governance can make deep ontology revision costly if migration rules are not made explicit

## Interpretation boundary
Scientific alignment is not the same thing as finished validation.
The research layer supports why the package is built this way. The pilot/testing layer is what still has to show how well it holds up in broader human use.

## What would reduce those residual risks
- holdout packet review so familiar and novel-item performance can be compared
- field-level confusion review across multiple users
- explicit interface-effect notes in pilot review
- open-world failure harvesting rather than only static adversarial examples
- documented ontology-change and migration rules before any deep schema edit
