# Baseline protocol memo v0.1

## Purpose
This memo defines the baseline condition for the first Route 1 pilot.

The baseline exists so the structured system is not evaluated in a vacuum.

## Baseline condition
Use **plain-language Route 1 descriptions** as the baseline.

Participant sees:
- a short natural-language description of a bodily-affective microstate

Participant must reconstruct:
- the Route 1 feature profile in a controlled response form

## Why this baseline first
This is the cleanest first comparison because:
- it is simple
- it uses ordinary language
- it avoids questionnaire bias in the first pass
- it directly tests whether the structured system changes reconstruction shape

## Baseline format rules
Descriptions should be:
- short
- concrete
- non-poetic
- no extra narrative
- no interpretation beyond the state itself

Good example:
> “A medium-intensity rising tight feeling in the chest, negative and reactive, with a freeze tendency.”

Bad example:
> “It felt like doom opened inside me and my soul folded.”
Too interpretive.

## Baseline trial count
Use the same number of baseline items as structured items.

Recommended first pilot:
- 5 structured
- 5 baseline

## Baseline scoring
Baseline responses should be scored with the same field-level rubric as structured responses.

That means the comparison is:
- not prose quality
- but reconstructed feature adequacy

## Future baseline extensions
Not required for first pilot, but possible later:
- checklist baseline
- clinician-interview style baseline
- minimal tag baseline

## Current recommendation
For pilot v0.1:
- use plain-language baseline only
- defer additional baselines until after first review
